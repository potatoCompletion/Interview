<전자문서 서비스>
1. REST API 설계/개발
-

기획부터 참여한 프로젝트 였기 때문에 좀 더 주인의식을 가지고 개발에 임할 수 있었던 것 같습니다.
사내 프레임워크를 이용해야 하고 MyBatis를 이용한 개발을 처음 진행해봐서 초기엔 어려움이 있었지만,
다행히도 금방 적응을 마칠 수 있어 작업을 해낼 수 있었습니다.
특이했던 점은 대외기관의 연동(KISA)을 말할 수 있습니다.
서버 구성, DB 설계, API 설계, 대외기관과의 연동 등 다양한 면에서 경험할 수 있었던 프로젝트라 배운 점이 많습니다.
특히 레디스를 통한 토큰 관리, DB 설계 시 고려할 점, 대외기관 연동 경험 에서 배운 점들이 많습니다.
동료분들과 힘을 합쳐 노력한 결과, 기한 내에 전자문서유통중계자 인증심사를 무사히 통과할 수 있었습니다.

2. 필요 DB 설계
- 기본적인 테이블 설계에 대해서 많이 익힐 수 있었습니다.
생각하지 못했던 부분들(테이블 관리를 위한 createDate, updateDate 컬럼 추가 등)을 선배 분들께 배우며
DB에 대해 많이 알 수 있었던 것 같습니다.

3. 토큰 인증을 이용한 로그인 기능 구현
- 토큰인증 기반의 로그인 기능을 구현했습니다. 더존 내부 프레임워크를 통해서 구현하였고, 레디스 서버에 저장하였습니다.
이 과정에서 리프레쉬 토큰 없이 액세스 토큰만 발급하는 단점을 발견했고, 이를 개선하기 위해 개인적으로 토큰 인증에 대한 개선점을 다시 공부했습니다.



--------------------------------


어떤 역할을 담당하였는가?
 - REST API 설계/개발, DB 개선, 토큰 인증 로그인 구현
서버는 어떻게 구성하였는가?
 - 모든 서버는 이중화 구조로 L4 스위치를 통해 분산되며 nginx 웹 서버와 was 서버, DB 서버, redis 서버, 키 관리 서버로 구성했습니다.
어떤 언어와 프레임워크, 라이브러리?
 - 자바 1.8, spring framework, logback
몇명의 사용자, 어느 정도의 트래픽
 - B2B 서비스이기 때문에 과도한 트래픽이 올거라고는 예상하지 않았습니다. 따라서 모든 서버를 이중화로만 구성했습니다.
해당 기술 스택을 사용한 이유
 - '위하고'라는 플랫폼에 종속되는 프로젝트 였기 때문에 사내 프레임워크를 이용해야 했습니다.
구조에 대한 설명 및 설계 관련 내용
 -
어떠한 알고리즘, 자료구조 혹은 디자인패턴을 사용하였는가?
개선점은 무엇이였나?
어떻게 프로젝트를 관리했고(스케줄, 소통 등) 어떻게 기여했는가?(문서화, 구조 설계, 개발 등)
자신의 프로젝트에서 다양한 상황을 가정해보자. (이런 경우 혹은 저런 경우엔 어떻게 하실 건가요? 같은 질문을 자주 함)
